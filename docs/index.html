What did we just see?

  • An arbitrary to generate random values
  • A property: \(\forall a,b \in \mathbb Z : add(a, b) = a + b\)

Any problem with this property?

add.ts


                        function add(a : number, b : number) {
                            return a + b;
                        }

add.test.ts


                        const property = forall(integer, integer, () => {
                            return add(a, b) === a + b;
                        }

We just reimplemented the function!

Other properties we could use

  • Identity: \(\forall a \in \mathbb Z : add(a, 0) = a = add(0, a)\)
  • Negation: \(\forall a \in \mathbb Z : add(a, -a) = 0 = add(-a, a)\)
  • Commutative: \(\forall a,b \in \mathbb Z : add(a, b) = add(b, a)\)
  • Associative: \(\forall a,b,c \in \mathbb Z : add(add(a, b), c) = add(a, add(b, c))\)
